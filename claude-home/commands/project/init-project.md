## Instructions

新プロジェクトを初期化する: **$ARGUMENTS**

1. **プロジェクト分析**: タイプとフレームワークを特定（引数なしの場合はユーザーに確認）
2. **基本構造**: ディレクトリ構造作成、git初期化、.gitignore、README.md
3. **フレームワーク設定**: 選択されたフレームワークに応じた設定ファイルを生成（TypeScript、lint、format等）
4. **テスト基盤**: テストフレームワーク設定、テストディレクトリ・初期テスト作成
5. **ビルド・開発ツール**: ビルドシステム、開発サーバー、環境変数管理を構成
6. **CI/CD**: GitHub Actions ワークフロー作成（テスト・デプロイ）
7. **検証**: 依存関係インストール、ビルド、テスト、lint/format、開発サーバー起動を確認し初回コミット
