## Instructions

新機能をスキャフォールドする: **$ARGUMENTS**

1. **計画**: 要件定義、受け入れ基準の策定、影響範囲の特定
2. **調査**: 既存コードベースのパターン・規約を把握、類似機能との一貫性を確認
3. **設計**: アーキテクチャ・データフロー設計、必要に応じてDB/APIスキーマ定義
4. **環境**: フィーチャーブランチ作成、必要な依存関係のインストール
5. **実装**: コア機能から段階的に構築、プロジェクトのコーディング規約に従う
6. **テスト**: ユニット・インテグレーション・E2Eテスト作成、エッジケース対応
7. **検証**: lint/format/build/test 通過を確認、セルフレビュー実施
