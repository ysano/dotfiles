## Instructions

プロジェクトタイムラインのシミュレーションと最適化を行う: **$ARGUMENTS**

Think harder about this analysis.

1. **コンテキスト把握**: プロジェクトスコープ、主要変数、リソース制約、リスク許容度をユーザーと確認（不明点があれば質問）

2. **プロジェクト構造モデリング**:
   - WBS（作業分解構造）分析
   - 依存関係マッピング（順序・並行・外部）
   - クリティカルパスの特定

3. **変数モデリング**: 工数推定、リソース、外部依存等の不確実性を3点見積もり（楽観・最頻・悲観）で定量化

4. **シナリオ生成**: ベースライン（40%）、楽観（20%）、悲観（20%）、ストレス（20%）の複数シナリオを生成

5. **リスク評価**: 技術・リソース・外部・スコープリスクの影響度と発生確率を評価、緩和策を提案

6. **リソース最適化**: チーム配置、予算制約下でのリソース配分を最適化

7. **レポート出力**: シナリオ比較、P50/P90完了日、クリティカルパス、リスク調整済みタイムライン、推奨アクションを含むレポートを生成
