## Instructions

プロジェクトの健全性を多角的に分析する。

Think harder about this analysis.

1. **初期化**: ツール接続確認（Linear, GitHub）、評価期間の設定（デフォルト30日）

2. **多次元分析**:
   - **コード健全性**: コミット頻度、変更頻度の高いファイル、ブランチ鮮度、テストカバレッジ
   - **依存関係**: 古い依存関係、セキュリティ脆弱性、ライセンス問題
   - **タスク管理**: スプリント速度トレンド、サイクルタイム、ブロック期間、バグ対機能比率
   - **チーム**: PRレビュー応答時間、コミット分布、作業バランス

3. **レポート生成**: 各カテゴリのスコア（/100）付き健全性レポートを出力
   - トレンド分析（速度・バグ発見率の推移）
   - リスク評価（高優先度リスクと緩和策）
   - アクションアイテム（即座・短期・長期）

4. **深掘りオプション**: 分析結果に基づき、速度低下・脆弱性修正・技術的負債等の深掘り分析を提案

## 前提条件

- Git リポジトリアクセス
- Linear MCP 接続（推奨、なくても git/GitHub メトリクスで部分分析可能）
