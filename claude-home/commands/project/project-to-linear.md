## Instructions

プロジェクト要件を分析し、Linear にタスク構造を同期する: **$ARGUMENTS**

1. **要件分析**: タスク説明/要件を分析、コードベース構造を把握、スコープと複雑さを評価
2. **ユーザー確認**: 目標、優先度、タイムライン、技術的制約について不明点があれば確認
3. **Linear 接続**: Linear MCP サーバーの利用可能性を検証（未設定なら設定手順を案内）
4. **プロジェクト設定**: 既存プロジェクト使用 or 新規作成をユーザーと決定
5. **タスク生成**: フェーズ別（計画・実装・テスト・ドキュメント・デプロイ）に詳細タスクを分解、依存関係と優先度を設定
6. **Linear 同期**: プロジェクト・親タスク・サブタスクを順に作成、依存関係・ラベル・優先度を設定
7. **サマリー**: 作成タスク数、フェーズ別内訳、クリティカルパス、Linear プロジェクトリンクを表示

## 注意事項

- Linear MCP サーバーが必要（未利用時はインポート可能な形式でタスク一覧を出力）
- レート制限を考慮したバッチ操作を使用
