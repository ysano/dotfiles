# 意思決定ツリー探索手順

複雑な意思決定シナリオを探索し、選択結果を最適化するための包括的な意思決定ツリー分析を作成する。

## 1. 前提条件評価

以下を検証:

- **意思決定範囲**: 具体的にどんな意思決定が必要か
- **ステークホルダー**: 誰が影響を受け、関与するか
- **時間制約**: 決定期限と実装タイムライン
- **成功基準**: 意思決定の成功/失敗をどう測定するか
- **リソース制約**: 利用可能な選択肢に影響する制限

## 2. 意思決定アーキテクチャマッピング

意思決定を体系的に構造化:

- 主要意思決定ポイントと中核的質問
- 主要選択から続く二次的意思決定
- 三次的意思決定と実装詳細
- 意思決定依存関係とシーケンス要件
- オプションの組み合わせと相互作用効果

ステークホルダー影響分析とドメイン制約を識別。

## 3. オプション生成と構造化

意思決定の代替案を体系的に特定:

- 目標達成への直接的アプローチ
- 複数アプローチを組み合わせるハイブリッドソリューション
- 段階的実装を伴うフェーズドアプローチ
- よりニーズに合う代替目標
- 比較用の「何もしない」ベースライン

各オプションについて実現可能性を評価:
- 技術的実現可能性
- 経済的実現可能性
- 組織的実現可能性
- 市場実現可能性

## 4. 確率評価フレームワーク

体系的な確率推定を適用:

- 類似意思決定の歴史的成功率
- 業界ベンチマークと比較データ
- 専門家判断とドメイン知識
- 市場調査と顧客検証データ
- 内部能力評価と実績

シナリオ確率の重み付け:
- ベストケース (楽観的結果)
- 最も可能性が高いケース (ベース期待)
- ワーストケース (悲観的結果)
- ブラックスワンイベント (極端シナリオ)

## 5. 期待値計算

意思決定結果を体系的に定量化:

- 財務リターンとコスト影響
- 戦略的価値と競争優位性
- リスク削減とオプション価値創造
- 時間節約と効率改善
- 学習価値と能力構築

期待値統合:
```
EV = Σ(確率 × 結果価値) すべてのシナリオについて
```

## 6. リスク分析と感度テスト

意思決定リスクを包括的に評価:

- 実装リスク (実行課題)
- 市場リスク (需要、競合、経済変化)
- 技術リスク (技術的実現可能性、陳腐化)
- 規制リスク (コンプライアンス、承認、政策変更)
- リソースリスク (可用性、能力、コスト超過)

感度分析:
- 主要前提のストレステスト
- 重要変数の損益分岐点分析
- パラメータ変動を伴うシナリオ分析
- 結果の信頼区間計算

## 7. 意思決定ツリー可視化

明確な意思決定ツリー表現を作成:

```
[意思決定ポイント]
├─ オプション A (確率: 60%)
│  ├─ 成功 (70%) → 結果: 300% ROI
│  └─ 失敗 (30%) → 結果: -50% ROI
│  期待値: 195%
└─ オプション B (確率: 40%)
   ├─ 成功 (40%) → 結果: 500% ROI
   └─ 失敗 (60%) → 結果: -80% ROI
   期待値: 152%

推奨: オプション A (リスク低、堅実リターン)
```

## 8. 最適化と推奨エンジン

データ駆動型の意思決定推奨を生成:

```markdown
## 主要推奨: [選択オプション]

### 根拠:
- 期待値: $X over Y years
- 戦略的コントロール: [利点]
- リスク評価: [レベル]
- タイムライン影響: [期間]

### 実装計画:
- フェーズ 1: [タイムライン]
- フェーズ 2: [タイムライン]
- フェーズ 3: [タイムライン]

### リスク緩和:
- [リスク]: [緩和戦略]
```

## 9. 意思決定品質検証

堅牢な意思決定プロセスを確保:

- すべての関連ステークホルダーに相談
- 包括的オプション生成完了
- データで較正された確率評価
- すべての重要要因を含む価値計算
- リスク特定と緩和計画
- 前提を明示的に文書化・テスト

バイアス検出:
- 確証バイアス
- アンカリングバイアス
- 可用性バイアス
- 楽観バイアス
- サンクコストの誤謬

## 10. 学習とフィードバック統合

意思決定品質の改善を確立:

- 実際 vs 予測結果の測定
- 実結果に対する前提検証
- 意思決定タイミングと実装効果
- ステークホルダー満足度とサポートレベル
- 意図しない結果と副作用
