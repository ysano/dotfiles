# 制約モデリング手順

意思決定環境を支配する制約を体系的にモデル化し、正確なシミュレーションとシナリオを作成する。

## 1. 前提条件評価

以下の要素を確認:

- **ドメイン定義**: どのシステム/環境の制約をモデル化するか
- **制約種別**: 物理的、経済的、規制的、技術的、社会的制約か
- **影響範囲**: 制約が意思決定と結果にどう影響するか
- **変化動態**: 制約は静的か時間とともに進化するか
- **検証ソース**: 制約の正確性を検証できるデータ/専門知識は何か

## 2. 制約分類

### ハード制約 (違反不可)
- 物理法則と自然的限界
- 地理的・空間的境界
- 法的・規制要件
- 安全性・セキュリティ要件

### ソフト制約 (管理/トレードオフ可能)
- 予算制限と財務リソース
- コスト最適化と効率目標
- 品質・パフォーマンス期待
- 時間的制約とスケジュール圧力

### 動的制約 (時間とともに変化)
- 技術進歩と陳腐化サイクル
- 市場成熟と顧客進化
- 競合動態と市場シフト
- 組織能力と学習曲線

## 3. 制約マッピングと可視化

制約関係モデルを作成:

- 主要制約 → 二次効果の分析
- 制約間の依存関係マッピング
- フィードバックループと相互作用の特定
- 制約階層のモデル化 (戦略 → 戦術 → 運用)

## 4. 前提検証フレームワーク

各制約について以下を文書化:

- 前提の説明と根拠
- 信頼度レベル (高/中/低)
- 検証方法とデータソース
- 感度分析 (前提が間違っていた場合の影響)
- 更新頻度とレビュースケジュール

歴史的検証と専門家相談を実施。

## 5. シナリオ境界定義

制約を使用して現実的なシナリオ限界を定義:

- 楽観的境界 (最良ケースの制約緩和)
- 現実的境界 (通常の制約条件)
- 悲観的境界 (最悪ケースの制約強化)
- 不可能領域 (制約違反するシナリオ)

ストレステスト: 最大制約負荷とブレークポイント。

## 6. 動的制約モデリング

時間経過に伴う制約変化をモデル化:

- 線形進化 (予測可能なペースで変化)
- 段階的変化 (規制変更等による急激なシフト)
- 周期的変動 (季節的・経済サイクル)
- 確率的変化 (ランダムイベントと不確実性)

## 7. 制約最適化戦略

制約内で機能し最適化するアプローチを生成:

- 直接緩和 (制約を変更・除去)
- 制約トレードオフ (ある制約を別の制約と交換)
- 制約再構成 (制約を有利に再定義)
- 創造的解決策 (制約を優位性に転換)

## 8. 出力生成

以下の形式でプレゼント:

```markdown
## 制約モデル分析: [ドメイン/プロジェクト名]

### 制約環境概要
- 主要制約カテゴリ
- 制約相互作用と依存関係
- 重大な制約リスクと機会

### 制約カタログ

#### ハード制約
1. [制約名]: [説明]
   - カテゴリ: [物理/規制/技術]
   - 影響: [高/中/低]
   - 検証ソース: [データ/専門知識]
   - 変更可能性: [固定/進化]

#### ソフト制約
[同様の形式]

### 制約最適化推奨
1. [制約名] への対処戦略
2. トレードオフ分析
3. リスク緩和アプローチ

### モニタリング計画
- 追跡すべき主要指標
- 更新頻度
- 早期警告トリガー
```

## 9. 継続的学習

- 実制約挙動 vs モデル予測のフィードバック統合
- 制約違反の教訓とリカバリー洞察
- ステークホルダーからの制約精度と完全性フィードバック
- 制約妥当性に影響する市場・環境変化
