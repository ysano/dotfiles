# 設計書: Issue/PR 番号 → 絵文字 ID（ハッシュ・2絵文字）

- 日付: 2026-06-25
- 状態: ドラフト（レビュー待ち）

## 目的・背景

ADHD 特性への対策として、Issue/PR 番号を決定論的に 2 つの絵文字へ変換し、**似た番号を視覚イメージで判別・記憶**できるようにする。連番（`#1234` / `#1235` / `#1243`）が一目で区別できることが主目的。

## 要件

- **決定論的・再現可能**: 同じ入力は、いつ・どこで・誰が計算しても同じ絵文字になる。スクリプト・Claude・他ツールで一致する。
- **似た番号の判別**: 連番・近接番号が視覚的に大きく異なること（ハッシュの雪崩効果）。
- **リポジトリ跨ぎの区別**: 別リポジトリの同一番号（`repoA#1` と `repoB#1`）は別の絵文字になること。
- **記憶しやすさ**: 覚える単位は 2 絵文字。

## 非対象（YAGNI）

- 絵文字からの逆引き（絵文字 → 番号）はしない。
- 絵文字の意味階層（1個目=種別 等）は持たせない。純粋な識別子とする。
- 番号規模による絵文字個数の自動可変はしない（常に 2 個固定）。

## アルゴリズム

1. **キー生成**:
   - 引数が `<repo> <number>` の 2 つ: `key = "${repo}#${number}"`（例 `"dotfiles#1234"`）
   - 引数が `<number>` のみ: `key = "${number}"`（フォールバック）
2. **ハッシュ**: `printf '%s' "$key" | shasum -a 256` → 64 文字の 16 進ダイジェスト
3. **インデックス化**:
   - `i = (先頭 2 hex を 10 進化, 0–255) % 64`
   - `j = (次の 2 hex を 10 進化, 0–255) % 64`
   - 256 は 64 の倍数のため剰余に偏りなし（各索引が等確率）
4. **パレット参照**: 固定パレット `P`（64 種）から `P[i]` と `P[j]` を連結
5. **出力**: `P[i]P[j]`（例 `🦊🍇`）。改行なしで stdout に出す。

### 衝突特性

- 空間は N² = 64² = 4,096 通り。同時に見比べる番号が数十でも誕生日問題による衝突はほぼ無視できる（数十番号で衝突確率 < 数%）。
- グローバルな一意性は保証しない（目的は"同時に見る少数の番号の判別"であり、全番号の一意採番ではない）。

## パレット（固定・順序がアルゴリズムの一部）

索引 0–63。順序を変更すると全出力が変わるため**変更不可**（変えるなら別バージョンとして扱う）。単一コードポイントの具象絵文字のみ（顔・肌色・国旗・異体字セレクタ必須・酷似記号は除外）。

## 実装

### スクリプト

- 配置: `bin/emoji-id`（新設のユーザーコマンド用ディレクトリ。管理用 `scripts/` とは役割分離）。実行可能・拡張子なし。手元で `emoji-id` として叩けるよう `.zsh` で既存ヘルパ `safe_path_append "$DOTFILES/bin"`（存在ガード付き＝グレースフル劣化）で PATH 追加。Claude は `bash bin/emoji-id …` で直接呼ぶ
- 言語: bash（`shasum` は macOS/Linux 共通。Linux で無ければ `sha256sum` にフォールバック）
- インターフェース:
  - `emoji-id 1234` → `🦊🍇`（番号のみ）
  - `emoji-id dotfiles 1234` → `…`（repo + 番号）
  - 引数なし/不正: usage を stderr に出し rc 1
- クロスプラットフォーム: `shasum -a 256` が無ければ `sha256sum` を使う分岐（グレースフル劣化）

### テスト

`test_emoji_id.sh`（`.tmux/claude/test_*.sh` と同様の素の assert スタイル）:

- **決定論**: 同一入力を 2 回呼び、出力が一致
- **既知値固定**: `emoji-id 1234` の出力が spec 記載の固定値と一致（リグレッション検出）
- **近接番号の差異**: `1234` / `1235` / `1236` が互いに異なる
- **repo 区別**: `emoji-id a 1` と `emoji-id b 1` が異なる
- **フォールバック**: `emoji-id 1`（repo 省略）が 2 絵文字を返す
- **パレット長**: パレットがちょうど 64 要素
- **異常系**: 引数なしで rc 1

### Claude の振る舞い・既存設定との整合

- `~/.claude/CLAUDE.md`「働き方の特性とサポート」に追記: PR/Issue 番号に言及するとき `emoji-id` の規則で 2 絵文字を併記する（例 `PR #1234 🦊🍇`）。
- `~/.claude/prompts/reply-style.md`: 「会話の装飾絵文字（原則不使用）」と「番号の**識別**絵文字」は別カテゴリと定義。識別絵文字は GitHub/JIRA でも番号の補助として可（ただし多用しない）。
